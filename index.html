第1回 WWW と HTML

ハイパーリンク（通称「リンク」）によって Web ページが接続されていて，ユーザはリンクを辿ってたくさんの Web ページを見ることができる．
Web ページではテキストに加えて，画像や音声，動画などマルチメディアコンテンツや PDF などのファイルにアクセスできる．

WWW について

ページを提供する仕組みのことを WWW という.
インターネット上には膨大な数の Web ページが存在する．その中からユーザが見たい Web ページを指定する方法として，URL がある．ブラウザのアドレス欄に表示されているのが URL である．URL にはプロトコル、サーバ、パスが含まれている．

HTML について

Web ページの構造を指定するための記述．ページは見出しや本文，箇条書き，図表などでできているが，このようなページの構造を記述するのが HTML (Hypertext Markup Language)である．Web ページを構成する「部品」の１つであり、他にはCSS, JavaScript, 画像・音声・動画がある．

演習の成果：演習1-4: HTMLのさまざまな要素

第2回 CSS によるWebページの装飾

CSS について

ページを装飾する記述を CSS という。CSS は見た目を変更するものである。
コーディングする順番は、HTML が先に与えられて、その装飾をする CSS を記述するという形となる。
まずはHTML の head要素に link要素を記述する。これはCSSファイルに従ってページを装飾するという意味を持つ。

CSS の書式について

  • セレクタ は装飾したい要素を指定
  • プロパティ名 は装飾の種類を指定
  • プロパティ値 は装飾の具体的な値
  • 「プロパティ名: プロパティ値」の記述は中括弧の中で繰り返し記述可能
  • 上記の書式全体を2つ以上記述可能

セレクタについて

CSS で装飾する対象要素を指定する記述.たくさんの書き方が存在する。

  • lass 属性は同じクラス名を2個以上の要素につけることができる
  • id 属性は，一つの HTML 文書の中で固有の id をつけなくてはならない（同じ id 属性を持つ要素が2つある状態は NG）

プロパティと値について

プロパティ名は様々あり，CSS においてあらかじめ定められた名前だけが使用できる．また，プロパティ値については，プロパティ名に対して許された値だけを指定できる．例えば，color に対しては色の名前か次に出てくるカラーコードだけが指定可能である．一方 font-size に対して，サイズを意味する値だけが指定可能である．

演習の成果：ウェブブラウザの紹介

第3回 JavaScript の基本

JavaScript について

JavaScript は Web ページ上で複雑な機能を実現するプログラミング言語である．
Web ページを開いた直後や，ボタンを押したときに JavaScript プログラムが実行される．
ページの自動更新やアニメーションなど動きのあるページを実現するために JavaScript プログラムを使う．

Java と JavaScriot について

基本的には Java と JavaScript は別の仕組みを持つ言語である．

最も大きな違いは，インタプリタ言語とコンパイル言語である．

Java はコンパイル言語である．Java プログラムを実行するには javac でコンパイルしたのち，java で実行するという2段階の手順を踏む必要がある．

一方，JavaScript はインタプリタ言語 である． JavaScript プログラムは，そのまま 1行ずつ読み取りながら実行される．Java におけるコンパイルという一手間がない．

インタプリタ型言語の JavaScript には次の特徴がある．

  • コンパイルの一手間がないため，プログラムを書いて実行するまでの作業が少なくて済む
  • 間違ったプログラムを実行すると実行時エラーが発生する．コンパイルエラーはない
  • 実行中に値の型が決まる、動的型付けがある

手軽に実行とデバッグを繰り返すことができるという長所に対して，実行時エラーや動的型付けにより，エラーが起こっていても気づかないことがあったり，エラーの原因を見つけるのが難しいという短所がある．

ブラウザ API について

Firefox などの標準的なブラウザは，JavaScript プログラムを通じて実行可能な便利な機能（API）を提供してくれる．
具体的には次のブラウザ API を利用することができる．

  • DOM API: プログラムを通じて HTML 要素や属性，CSS などの追加・編集・削除を可能にしてくれる
  • Geolocation API: プログラムを通じて 現在位置などを取得できます
  • Canvas と WebGL: 2D と 3D のグラフィックスのプログラミングを可能にします
  • WebRTC: リアルタイムに通信を行ったり，ビデオストリームを使った遠隔会議を実現できます

サードパーティ API について

インターネット上には JavaScript プログラムからの通信を受け付けるサーバがある．
通信によってサーバが持つ情報を提供してくれたり，サーバ上に情報を登録してくれたりする．
このような Web サービスの例を示す．

  • Twitter API: ツイートを検索し取得できる
  • Google Map API: Web ページにマップを貼り付けることができる
